# Project Bootstrap Checklist

Use this checklist when instantiating the OS in a new project repo.

## Bootstrap Checklist
- Copy `AGENT_MODEL.md`
- Copy `KADE.md`
- Copy `MASON.md`
- Copy `LEDGER.md`
- Copy `CASE.md`
- Copy the selected process docs into the project repo
- Copy the selected templates into the project repo
- Instantiate the project's documentation map
- Instantiate the project's current state artifact
- Instantiate the project's decisions log
- Instantiate the project's drift log
- Instantiate the project's worklog
- Instantiate the first Sprint Spec
- Instantiate the first Technical Implementation Plan
- Instantiate the first Acceptance / Test Plan
- Instantiate the Sprint Governance Review record/checklist
- Confirm contributors know where the active project artifacts live

## Ready-To-Execute Check
The project is ready for sprint implementation only when:
- the role entry files are in place
- the project-local artifact set exists
- the active sprint has an instantiated Technical Implementation Plan
- the active sprint has an instantiated Acceptance / Test Plan
- the active sprint has an instantiated Sprint Spec

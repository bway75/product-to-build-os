# Handoff Message Templates

Use these as short manual transfer contracts between roles.

## Brian / Kade -> Mason
- package ready: yes/no
- artifacts included:
  - Product Brief / 6-Pager
  - PRD
  - JTBD / User Stories
  - Technical Implementation Plan
  - Acceptance / Test Plan
  - Sprint Spec
- open risks:
- expected next action: Mason begins implementation / asks clarifying questions
- escalation needed: yes/no

## Mason -> Case
- package ready: yes/no
- artifacts included:
  - Sprint Spec
  - Acceptance / Test Plan
- implementation summary:
- verification results:
- open risks / known limitations:
- expected next action: Case starts validation
- escalation needed: yes/no

## Case -> Ledger
- package ready: yes/no
- artifacts included:
  - Sprint Spec
- Case validation summary:
- drift / deviation notes:
- docs/process follow-up context:
- expected next action: Ledger performs Sprint Governance Review
- escalation needed: yes/no

## Escalation To Kade
- package ready: yes/no
- issue summary:
- impacted role:
- blocking risk:
- artifacts reviewed:
- expected next action from Kade:

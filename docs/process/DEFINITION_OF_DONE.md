# Definition Of Done

Minimum sprint-closeout standard for teams using this OS.

## Functional
- sprint acceptance criteria met
- no unapproved scope drift
- Technical Implementation Plan and Acceptance / Test Plan remained aligned with delivered scope, or updates were made explicitly
- the sprint used an instantiated Technical Implementation Plan and Acceptance / Test Plan, not just template files

## Quality
- the project's declared verification baseline passes
- new application behavior includes appropriate tests when practical

## Governance
- Case validation completed against the instantiated Acceptance / Test Plan
- Sprint Governance Review completed
- required project-local artifacts updated as needed
- drift recorded if product-impacting deviation occurred
- decisions log updated when a design-authority boundary was crossed

## Repo Health
- working tree clean or intentionally understood per project workflow
- committed and synced per the project's chosen branch policy

# Artifact Readiness Checklists

Use these lightweight checks before a role starts work.

## Before Mason Starts
- Product Brief / 6-Pager exists
- PRD exists
- JTBD / User Stories exist when used by the project
- Technical Implementation Plan is instantiated
- Acceptance / Test Plan is instantiated
- Sprint Spec is instantiated
- constraints / exclusions are explicit
- open risks / escalation notes are visible

## Before Case Starts
- Sprint Spec is available
- Acceptance / Test Plan is instantiated
- implementation summary is available
- verification results are available
- drift / deviation notes are available if needed
- known limitations are visible

## Before Ledger Starts
- Sprint Spec is available
- implementation summary is available
- verification results are available
- Case validation summary is available
- drift notes are available if drift occurred
- docs/process follow-up context is available
